# MLOps 프로젝트 체크리스트

> 이 체크리스트는 평가 기준이 아닌 **진행 가이드**입니다.
> 단계마다 스스로 점검하면서 활용하세요.

---

## ☑ 문제 정의

- [ ] 해결하려는 문제가 명확한가?
- [ ] 입력과 출력이 정의되어 있는가?
- [ ] 성공 기준 또는 평가 지표가 있는가?

---

## ☑ 데이터

- [ ] 데이터 출처와 구조를 설명했는가?
- [ ] 전처리 과정을 코드로 재현할 수 있는가?
- [ ] train / validation / test 분리가 되어 있는가?
- [ ] 데이터 누수(data leakage)가 없는가?
  - scaler, encoder 등을 train으로만 fit했는가?
  - test 데이터를 전처리 과정에 사용하지 않았는가?

---

## ☑ 모델링

- [ ] Baseline 모델이 있는가?
- [ ] 모델 선택 이유를 설명할 수 있는가?
- [ ] 실험 결과를 `experiments/` 폴더에 기록했는가?
- [ ] 성능 지표를 해석했는가?
- [ ] 시드가 고정되어 있는가? (`random`, `numpy`, `torch`)

---

## ☑ 배포

- [ ] 사용자가 입력을 넣고 결과를 받을 수 있는가?
- [ ] Streamlit 등으로 외부에서 실행 가능한 데모가 있는가?
- [ ] 실행 방법이 README에 정리되어 있는가?
- [ ] 모델 크기가 Streamlit Cloud 제약(~1GB RAM) 내인가?
- [ ] CPU 환경에서도 동작하는가?

---

## ☑ 모니터링

- [ ] 예측 요청이나 결과 로그를 남기는가?
- [ ] latency, error, input example 등 최소한의 운영 정보를 기록하는가?
- [ ] 모델의 한계나 성능 저하 가능성을 설명했는가?
- [ ] `logs/` 폴더가 `.gitignore`에 포함되어 있는가?

---

## ☑ 협업

- [ ] Issue, branch, PR을 사용했는가?
- [ ] 브랜치 네이밍 컨벤션을 따르는가? (`feature/`, `fix/`, `experiment/`)
- [ ] 커밋 메시지 규칙을 따르는가? (`[feat]`, `[fix]`, `[data]` 등)
- [ ] 팀원별 역할과 기여가 정리되어 있는가?
- [ ] secrets이 GitHub에 커밋되지 않았는가?

---

## ☑ 발표

- [ ] 문제 → 데이터 → 모델 → 배포 → 모니터링 흐름이 연결되어 있는가?
- [ ] 데모 화면 또는 실행 예시가 있는가?
- [ ] 한계와 개선 방향을 설명하는가?
- [ ] AI 도구 사용 내역을 정리했는가?

---

## 발표 자료 권장 구성

1. 문제 정의
2. 데이터 소개
3. 전체 아키텍처
4. 모델링 방법 및 실험 결과
5. 배포 데모 (라이브 or 스크린샷)
6. 모니터링 / 운영 관점
7. 한계와 개선 방향
