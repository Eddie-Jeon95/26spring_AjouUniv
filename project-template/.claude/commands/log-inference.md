# /log-inference

배포된 모델의 운영 로그를 분석하고 모니터링 상태를 점검하기 위한 커맨드입니다.

---

`logs/inference.jsonl`의 내용을 분석해주세요.

## 1단계: 로그 파일 읽기
- `logs/inference.jsonl` 파일을 읽고 전체 레코드 수 확인
- 파일이 없거나 비어 있으면 로깅 코드가 연결되어 있는지 확인 후 알려주세요

## 2단계: 기본 지표 계산

다음 항목을 계산해서 표로 보여주세요:

| 지표 | 값 |
|------|-----|
| 총 요청 수 | |
| 성공 요청 수 | |
| 에러 요청 수 (에러율 %) | |
| 평균 latency (ms) | |
| 최대 latency (ms) | |
| P95 latency (ms) | |

## 3단계: 이상 감지

다음 기준으로 이상 징후가 있는지 확인해주세요:
- latency가 평균의 3배 이상인 요청
- 연속으로 에러가 발생한 구간
- 입력 길이나 형식이 다른 요청 (있다면 예시 표시)

## 4단계: 요약 및 제안

```
[현재 상태]
- 정상 / 주의 / 위험 중 하나로 판단

[눈에 띄는 패턴]
- (있다면 기술)

[권장 액션]
- (있다면 기술)
```

## 5단계: CLAUDE.md 업데이트 제안
발견한 이상 패턴이나 운영 팁이 있으면 CLAUDE.md 7번 섹션(알려진 함정)에 추가할 내용을 제안해주세요.
(직접 수정하지 말고 제안만)

---

⚠️ **주의**:
- 로그에 개인정보(이름, 이메일 등)가 포함된 경우 내용을 출력하지 말고 존재 여부만 알려줄 것
- 분석은 요약 수준으로만, 원본 로그를 그대로 출력하지 말 것
