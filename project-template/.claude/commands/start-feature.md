# /start-feature

새 기능을 구현하기 전에 단계적으로 접근하기 위한 커맨드입니다.

---

$ARGUMENTS 기능을 구현하려고 합니다.

**바로 코드를 작성하지 말고, 다음 순서로 진행해주세요.**

## 1단계: 컨텍스트 파악
- CLAUDE.md를 먼저 읽고 프로젝트 컨벤션 확인
- 이 기능과 관련된 기존 코드/파일을 검색해서 읽기
- 영향 받을 파일 목록 정리

## 2단계: 구현 계획 제시

```
[기능 요약]
이 기능은 ___을 위해 ___을 합니다.

[영향 받는 파일]
- src/xxx/yyy.py (수정)
- src/xxx/zzz.py (신규)

[구현 단계]
1. ___
2. ___
3. ___

[검증 방법]
- 어떻게 동작을 확인할지
- 어떤 엣지 케이스를 테스트할지

[질문/확인 필요사항]
- 애매한 부분이 있다면 여기에 질문
```

## 3단계: 승인 대기
**계획만 보여주고 멈춰주세요.** 검토 후 진행 여부를 알려드립니다.

## 4단계: 단계별 구현
승인되면 위 계획의 1번부터 하나씩 구현합니다.
- 한 단계가 끝날 때마다 변경사항 요약
- 다음 단계로 넘어가기 전 확인을 기다림
- 실행/테스트 방법도 함께 제안
