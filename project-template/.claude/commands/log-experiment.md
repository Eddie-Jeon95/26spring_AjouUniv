# /log-experiment

ML 실험 결과를 표준 형식으로 기록하기 위한 커맨드입니다.

---

방금 진행한 실험을 `experiments/` 폴더에 기록해주세요.

## 1단계: 정보 수집
다음을 먼저 확인:
- 어떤 실험을 했는가 (학습 코드, config, 데이터)
- 어떤 결과가 나왔는가 (메트릭, 학습 곡선, 출력 예시)
- CLAUDE.md의 "현재 진행 상황"과 어떻게 연결되는가

부족한 정보가 있으면 질문해주세요.

## 2단계: 폴더 생성
`experiments/YYYYMMDD_실험명/` 형식으로 생성.
예: `experiments/20260425_lgbm_baseline/`

## 3단계: result.md 작성

```markdown
# 실험: [실험명]

**날짜**: YYYY-MM-DD  
**작성자**: [이름]  
**관련 이슈/PR**: #xx

## 가설
- 무엇을 검증하려 했는가?
- 예상한 결과는?

## 실험 설계
- 데이터: [어떤 데이터, 크기, split 비율]
- 모델: [구조, 주요 하이퍼파라미터]
- 학습 환경: [Colab T4 / 로컬 등, 학습 시간]

## 결과
| 메트릭 | 값 | 이전 baseline |
|--------|-----|--------------|
| ___    | ___ | ___          |

## 해석
- 가설과 결과가 일치했는가?
- 예상과 다른 부분이 있다면 왜?

## 다음 액션
- [ ] 시도해볼 것 1
- [ ] 시도해볼 것 2

## 재현 방법
\`\`\`bash
python scripts/train.py --config experiments/20260425_lgbm_baseline/config.yaml
\`\`\`
```

## 4단계: 부속 파일 복사
- 사용한 config 파일을 같은 폴더에 복사
- 모델 가중치는 용량이 크면 경로만 기록 (HF Hub 링크 등)

## 5단계: CLAUDE.md 업데이트 제안
"현재 진행 상황" 섹션에 반영할 내용이 있으면 알려주세요.
(직접 수정하지 말고 제안만)

---

⚠️ **주의**:
- 탐색적 실험도 기록. 가설 칸에 "탐색적 실험"이라고 명시.
- 실패한 실험도 반드시 기록 (오히려 더 가치 있음).
