# /review-pr

PR을 올리기 전 셀프 리뷰를 위한 커맨드입니다.

---

지금까지 작업한 내용을 PR로 올리기 전에 셀프 리뷰해주세요.

## 1단계: 변경사항 파악
- 현재 브랜치의 변경된 파일 목록과 핵심 변경점 요약

## 2단계: 체크리스트 검토

### 📋 일반
- [ ] CLAUDE.md 업데이트가 필요한 변경인가? (폴더 구조, 새 의존성, 컨벤션 등)
- [ ] README의 실행 방법이 여전히 유효한가?
- [ ] requirements.txt에 새 의존성이 모두 명시되었는가?

### 🔐 보안/secrets
- [ ] API 키, 비밀번호 등이 코드에 하드코딩되지 않았는가?
- [ ] `.env`, `secrets.toml` 같은 파일이 커밋에 포함되지 않았는가?

### 🤖 ML 코드 (해당 시)
- [ ] 데이터 누수 가능성은 없는가? (test 데이터로 전처리 fit, target leakage 등)
- [ ] 시드가 고정되어 있는가?
- [ ] 평가 메트릭이 문제 정의에 맞는가?

### 📊 운영 로그 (배포 관련 변경 시)
- [ ] 추론 결과가 `logs/inference.jsonl`에 기록되는가?
- [ ] 로그에 timestamp, request_id, latency_ms, status가 포함되는가?
- [ ] `logs/` 폴더가 `.gitignore`에 추가되어 있는가?

### 🚀 배포 관련 (해당 시)
- [ ] 모델 크기가 Streamlit Cloud 1GB 제약 안에 들어가는가?
- [ ] CPU 추론으로 동작하는가?

## 3단계: PR 메시지 초안 작성
`.github/PULL_REQUEST_TEMPLATE.md` 양식에 맞게 작성해주세요.

## 4단계: 결론
- ✅ 이대로 PR 올려도 좋습니다
- ⚠️ 다음 항목을 먼저 수정 추천: [목록]
- ❓ 다음 부분이 애매합니다, 확인 필요: [목록]

---

⚠️ **주의**:
- 직접 코드를 수정하지 말고 **지적만** 해주세요
- 사소한 스타일 이슈(변수명 취향 등)는 넘어가도 됩니다
