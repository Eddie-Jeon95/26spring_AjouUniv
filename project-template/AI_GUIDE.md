# Claude / AI 코딩 도구 활용 가이드

> AI 도구는 "대신 코딩하는 도구"가 아닙니다.
> **개발 생산성을 높이는 보조 도구**로 활용하세요.

---

## 권장 사용 방식

- 전체 프로젝트를 한 번에 생성시키지 않는다
- **기능 단위로 작은 작업**을 요청한다
- 현재 파일 구조와 제약 조건을 함께 제공한다
- 생성된 코드는 반드시 **직접 실행하고 이해**한다
- 불필요한 라이브러리 추가 여부를 확인한다
- 보안 정보나 API key를 절대 입력하지 않는다

---

## 슬래시 커맨드 활용

Claude Code에서 다음 커맨드를 사용하면 표준화된 방식으로 작업할 수 있습니다.

| 커맨드 | 용도 | 언제 사용 |
|--------|------|-----------|
| `/start-feature [기능명]` | 새 기능 단계별 구현 | 새 기능 시작 전 |
| `/log-experiment` | 실험 결과 표준 기록 | 모델 학습/실험 후 |
| `/log-inference` | 운영 로그 분석 | 배포 후 주기적으로 |
| `/review-pr` | PR 올리기 전 셀프 리뷰 | PR 작성 전 |

## 작업 요청 템플릿

AI에게 요청할 때 아래 형식을 참고하세요.

```markdown
## Goal
이번 작업의 목표는 다음과 같습니다.
[목표를 한 문장으로]

## Current Context
현재 프로젝트 구조와 관련 파일은 다음과 같습니다.
[현재 파일 구조나 관련 코드 스니펫]

## Constraints
- 기존 폴더 구조를 유지하세요.
- 불필요한 라이브러리는 추가하지 마세요.
- 변경한 이유를 설명하세요.
- 실행 방법과 테스트 방법을 함께 제안하세요.

## Task
다음 기능을 구현하거나 수정하세요.
[구체적인 작업 내용]

## Expected Output
- 수정 파일 목록
- 변경 요약
- 실행 방법
- 테스트 방법
```

## AI 사용 시 주의사항

### 코드 검증
- AI가 생성한 코드는 **반드시 직접 실행**해서 확인
- 이해하지 못하는 코드는 **설명을 요청**한 뒤 사용
- 에러가 나면 에러 메시지 전체와 **어디서 발생했는지** 맥락을 함께 제공

### ML 특이사항
- 데이터 전처리 코드에서 **train/test 분리** 순서를 특히 주의
- AI가 제안한 평가 메트릭이 **문제 정의에 맞는지** 확인
- 시드 고정 코드를 AI가 빠뜨리는 경우가 있으니 직접 확인

### 보안
- API key, DB 비밀번호 등은 절대 AI에 입력하지 말 것
- `.env` 파일을 AI에 보여주지 말 것
