# MLOps 프로젝트 템플릿 설명

아주대학교 2026년 봄학기 AI 수업 조교 준비 자료

---

## 무엇을 만들었는가

학생들이 복사해서 바로 쓸 수 있는 **프로젝트 템플릿**입니다.
코드를 대신 짜주는 게 아니라, **스스로 완성할 수 있는 구조와 흐름을 미리 잡아놓은 것**입니다.

학생 배포 대상은 `project-template/`이며, 이 설명 문서는 조교/관리자가 템플릿 의도를 이해하고 유지보수하기 위한 문서입니다.
Git 기준 배포에서는 `.gitignore`에 잡힌 데이터, 모델, 로그, 실행 산출물이 포함되지 않습니다.

---

## 핵심 설계 철학

**"학생과 Claude Code가 같은 폴더 규칙을 보고 움직이게 했다"** 가 핵심입니다.

`CLAUDE.md`를 Claude Code 작업 기준의 단일 문서로 두고, `docs/specs/`는 문제 정의, 데이터 분석, 모델링, 지표 해석의 판단 기준을 잡습니다.
학생들이 어떤 파일을 어디에 남길지 덜 고민하고, 데이터 분석과 실험 해석에 집중하도록 만드는 구조입니다.

> "End-to-End"는 자동화 파이프라인이 아니라, **전 과정을 직접 경험**한다는 의미입니다.
> 학생들이 데이터 전처리 → 모델링 → 배포 → 모니터링을 직접 손으로 해보며 전체 흐름을 이해하는 것이 목표입니다.

## 개발 사이클 상세

```
문제 정의
    │
    ▼
PROJECT_SPEC.md에 입력/출력, metric, Streamlit 최종 기준 작성
    │
    ▼
EDA와 데이터 카드
    │  • data/raw/
    │  • notebooks/
    │  • reports/DATA_CARD.md
    │  • data_manifest.json
    │
    ▼
전처리와 processed 데이터
    │  • scripts/preprocess.py
    │  • data/processed/
    │  • data_manifest.json
    │
    ▼
Baseline 학습과 실험 기록
    │  • scripts/train.py
    │  • experiments/runs/
    │  • model_registry.json
    │
    ▼
결과 해석과 추가 실험
    │  • metrics.json
    │  • predictions.csv
    │  • confusion_matrix.json
    │  • reports/EXPERIMENT_REPORT.md
    │  • reports/ERROR_ANALYSIS.md
    │
    ▼
Streamlit 실험/로그 대시보드
       • streamlit_app.py
       • src/utils/logger.py
       • logs/inference.jsonl
```

## 프로젝트 단계별 가이드 위치

| 단계 | 학생이 할 일 | 참고 위치 |
|------|------------|-----------|
| **처음 시작** | 전체 진행 순서 확인 | `README.md` |
| **Claude Code 사용** | 작업 기준, 폴더 규칙, 금지사항 확인 | `CLAUDE.md` |
| **문제 정의** | 입력/출력, metric, Streamlit 최종 기준 작성 | `docs/specs/PROJECT_SPEC.md` |
| **환경 세팅** | requirements.txt, .env 설정 | `README.md` §6, `.env.example` |
| **EDA** | `/eda data/raw/[파일명] target=[target]`로 계획 + 노트북 실행 | `notebooks/`, `reports/DATA_CARD.md` |
| **데이터 전처리** | `/preprocess-data ...`로 processed CSV 생성 | `scripts/preprocess.py`, `data/processed/` |
| **모델링** | `/train-baseline ...`로 baseline 학습 | `scripts/train.py`, `experiments/runs/` |
| **학습 실행** | CLI 인자로 processed 파일과 target 전달 | `scripts/train.py` |
| **실험 기록** | `/log-experiment` | `experiments/` |
| **배포** | streamlit_app.py 완성 | `streamlit_app.py`, `.streamlit/config.toml` |
| **결과 시각화** | metric 추이, confusion matrix, 로그 지표 연결 | `streamlit_app.py`, `experiments/runs/`, `logs/` |
| **모니터링** | 로그 연결 + 주기적 점검 | `src/utils/logger.py`, `/log-inference` |
| **최종 점검** | 체크리스트 자가 점검 | `MLOPS_CHECKLIST.md` |

## 슬래시 커맨드가 심어주는 습관

| 커맨드 | 사용 시점 | 심어주는 습관 |
|--------|-----------|--------------|
| `/eda data/raw/... target=...` | raw 데이터 확보 후 | 먼저 질문을 세우고 EDA 결과를 Data Card에 연결 |
| `/preprocess-data ...` | EDA 후 baseline 전 | raw와 processed를 분리하고 fit 기반 전처리를 피함 |
| `/train-baseline ...` | processed 데이터 준비 후 | 학습 전 계획과 학습 후 기록을 한 흐름으로 연결 |
| `/plan-experiment [실험 아이디어]` | 새 모델/feature 실험 전 | 가설, 비교 기준, 실패 조건을 먼저 정리 |
| `/log-experiment` | 모델 학습/실험 후 | 실패한 실험도 기록, 재현 가능한 실험 관리 |
| `/compare-models` | 여러 run 비교 | 같은 data_version끼리 metric과 trade-off 비교 |
| `/analyze-errors` | 성능 개선 고민 | 오답 유형과 다음 실험을 연결 |
| `/log-inference` | 배포 후 주기적으로 | 배포 후에도 모니터링하는 시각 |

> Claude Code를 쓰되, **학생이 검토하고 판단하는 루프**를 명시적으로 설계한 것이 특징입니다.
> 특히 실험은 metric 숫자뿐 아니라 confusion matrix, 오답 샘플, 추론 로그까지 함께 보도록 설계했습니다.
